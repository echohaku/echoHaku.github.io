Git 系列分享 - p.15 `git flow`

November 25, 2025
#Git

在前面幾篇，我們掌握了如何使用 git clone 取得程式碼，並透過 git pull 和 git push 進行同步。然而，當一個團隊同時開發數十個功能時，如果每個人都直接在 main 或 master 分支上操作，專案歷史將會一團混亂。

本篇將介紹業界最常用的結構化開發模型—— Git Flow，以及它如何搭配審核機制 Pull Request (PR) 來確保程式碼品質與穩定性。

一、Git Flow：結構化的分支管理模型

定義： Git Flow 通過定義五種不同角色的分支，來規範程式碼從開發、整合、發佈到修復的整個生命週期。

分支名稱角色定位發生情境相關指令/操作
main主要分支 (Production Ready)永遠保持穩定、可部署的狀態。所有正式版本標籤（Tag）都打在這裡。只能從 release 或 hotfix 合併進來，不會在此分支上做commit！
develop整合分支 (Integration Branch)所有開發分支的基準，從此分支開出feature ，開發完後也都會合併到這裡。
feature/<name>功能分支 (Feature Branch)開發新功能時，從 develop 分支出來，完成後再合併回 develop。git checkout -b feature/new-login
release/<ver>發佈分支 (Release Branch)準備發佈新版本時，從 develop merge到release，進行最終 Bug 修復和版本號更新。git checkout -b release/1.0.0 develop
hotfix/<name>熱修分支 (Hotfix Branch)當產品上線後發現 main 分支有緊急 Bug 時，直接從 main 分支出來修復，修復後須合併回 main 和 develop。

Git Flow 的優點

  • 隔離風險： 確保 main 始終是穩定的，所有不穩定的程式碼都在 feature 或 develop 上進行測試。
  • 版本明確： release 和 hotfix 使得版本發佈和緊急修復的流程非常清晰。

二、Pull Request (PR)：變更的審核與整合

Pull Request（或 Merge Request，視遠端服務而定）是一種在遠端代管平台（如 GitHub）上發起的協作機制。

1. 定義

PR 本質上是向專案管理者提出一個請求 (Request)，請求他們將你某個分支（例如 feature/new-login）的變更拉取 (Pull) 到目標分支（例如 develop 或 main）中。

2. 發生情境與流程

  • 情境： 你在自己的 feature 分支上完成了所有工作，並已推送到遠端（git push origin feature/my-work）。現在需要讓團隊審查你的程式碼，並將其整合進主線。
  • PR 的標準生命週期：
    1. Commit & Push： 在本地完成工作並推送到遠端分支。
    2. Create PR： 在 GitHub 網站上，從你的分支（Source Branch）向目標分支（Target Branch，如 develop）創建 PR。
    3. Code Review： 審查人員（Reviewer）檢查你的程式碼、測試、文件，並提供評論或建議修改。
    4. Iteration： 如果有修改要求，你在本地進行修改、Commit、並 git push（PR 會自動更新）。
    5. Approval & CI/CD： 獲得足夠的核准（Approval），且通過所有自動化測試 (CI/CD)。
    6. Merge： 專案管理者將 PR 合併到目標分支，然後關閉該 PR。

3. 知識點：PR 的重要性

  • 程式碼品質： PR 是確保所有新程式碼都經過至少一個隊友審核的唯一途徑，有效減少 Bug。
  • 知識分享： 隊友透過 PR 了解彼此的程式碼，促進知識交流。
  • 流程控制： 確保只有經過審核和測試的程式碼才能進入主要的 develop 或 main 分支。

三、小結

在一個遵循 Git Flow 的團隊中，你的日常工作流程將會是：

  1. 從 develop 分支出 feature/X。
  2. 在 feature/X 上開發、Commit。
  3. git push origin feature/X。
  4. 發起 PR，請求合併至 develop。
  5. 審核通過後，合併（Merge）進入 develop。
  6. develop 累積足夠功能後，分支出 release/Y 進行發佈準備。

Git 系列分享 - p.14 啟動協作：git fork 與 git clone 的差異

November 20, 2025
#Git

在Git 系列分享 - p.13 遠端協作的橋樑：git remote、git pull 與 git push中，我們學會了如何使用 git remote、git pull 和 git push 在本地和遠端倉庫之間傳輸變更。

今天我們將回到起點，探討在開始任何協作之前，取得專案程式碼的兩種最主要方式：git clone 與 git fork。許多初學者常將兩者混淆，但它們的作用層次和目的截然不同。

一、git clone：遠端倉庫的「本地副本」

git clone 是最直接的 Git 指令，它的任務只有一個： 把遠端的整個 Git 倉庫完整下載到你的本地電腦。

1. 定義

  • git clone 會把遠端倉庫的 所有分支、所有 Commit、所有歷史紀錄 完整複製到本地，並建立對應的工作目錄。
  • 執行後，Git 會自動替你設定一個名為 origin 的遠端連結，指向你 clone 的來源。

2. 發生情境與指令

情境作用說明範例指令
首次取得專案第一次開始一個新專案，需要將程式碼抓到本地電腦上。git clone https://github.com/user/project.git
團隊內部協作你在一個有寫入權限的團隊專案中工作。git clone git@github.com:team/project.git
查看遠端連結clone 後，Git 已自動配置好 origin。git remote -v

二、git fork

git fork 不是一個 Git 指令，而是 GitHub、GitLab 等遠端代管服務提供的功能。它的目的是在伺服器上（例如 GitHub 網站）創建一個與原倉庫完全獨立的副本。

1. 定義與比喻

與 clone 不同，fork 不是 Git 指令，而是 GitHub、GitLab 等平台提供的功能。
它的作用是在遠端服務器上建立一個你自己的、與原倉庫相互獨立的副本。

  • 定義： Fork 會在遠端平台（如 GitHub）上建立一個屬於你帳號的完整複製倉庫。你擁有這個分身的完整寫入權限，但它與原始倉庫沒有直接權限關係。 當你看到別人的專案想修改，但沒有權限時，可以先 fork 一份到自己的 GitHub，再用 git clone 把它下載到電腦進行修改。

2. 發生情境與流程

情境作用說明流程
貢獻開源專案你想貢獻一個你不具備寫入權限的開源專案。1. 在 GitHub 網站上點擊 Fork 按鈕。
保護主線作為保護主線倉庫的方法，所有外部貢獻都必須透過你的 Fork。2. git clone 你的 Fork 版本到本地。
整合變更將你的變更提交回原始倉庫。3. 在本地推送（push）到你的 Fork。
提出貢獻提供你的修改給原作者。4. 向原始倉庫發起 Pull Request (PR)。

三、核心差異與關係

特性git clonegit fork
操作層次本地操作（Git 指令）遠端操作（代管服務功能）
結果產生一個本地工作目錄。產生一個遠端倉庫分身。
權限關係需要對來源倉庫有讀取權限。你擁有 Fork 倉庫的完整寫入權限。
主要目的建立工作環境，進行本地開發。建立貢獻管道，準備發起 Pull Request。
協作順序通常發生在 fork 之後（fork $\rightarrow$ clone）。發生在貢獻之前（在網站上先 Fork）。

四、如果你是從本地開始：用 git remote add 綁定遠端

在某些情況下，你可能是：

  • 先在本地建立了一個新專案（git init）
  • 撰寫了一些程式碼
  • 之後才在 GitHub 上建立空倉庫

這時就不能用 clone，而是要手動綁定遠端倉庫。

1. 初始化本地倉庫

git init 
# 1. 初始化一個本地 Git 倉庫
# (新增檔案、git add、git commit)

2. 使用 git remote add 建立連結

使用 git remote add 指令，將遠端 URL 綁定到一個短名（通常是 origin）：

# 語法： git remote add <短名> <遠端倉庫URL>
git remote add origin https://github.com/your-username/your-new-project.git

# 驗證連結是否成功 (會顯示 fetch 和 push 的 URL)
git remote -v

3. 首次推送（Push）

在建立了遠端連結後，你就可以將本地的 main 分支推送到遠端 origin 倉庫，並建立追蹤關係：

# 首次推送並設定追蹤關係 (-u 旗標很重要)
git push -u origin main
  • -u (或 --set-upstream)：這個旗標會告訴 Git，將本地的 main 分支與遠端的 origin/main 分支建立連結。未來你就可以直接使用 git pull 或 git push，而無需指定 origin main。

小結

指令/操作作用位置目的
git fork遠端服務器（如 GitHub）建立獨立的遠端分身，以取得寫入權限和貢獻管道。
git clone本地電腦（Git 指令）將遠端或 Fork 倉庫完整複製到本地工作區。
git remote add本地電腦（Git 指令）將本地專案與遠端倉庫綁定，建立上傳通道。

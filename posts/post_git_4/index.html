Git 系列分享 - p.2 Git 對於修改的文件是怎麼劃分的呢？

October 3, 2025
#Git

如同 Git 系列分享Day 1 - Git是什麼？ 所述，Git 是一種強大的版本控制工具。然而，Git 並不會將你所有的修改自動納入版本紀錄。

要理解 Git 如何追蹤和儲存你的工作，必須先了解它將一個專案劃分為的三個核心工作區域（Three States）。

Git 的三個工作區域與主要流程

Git 的版本控制流程可以被簡化為三個步驟，文件會依序在三個區域間流動：

工作目錄 →（git add）→ 暫存區 →（git commit）→ Git 倉庫

這三個區域代表了 Git 對你本次修改所處狀態的認定：

  1. 🗂️ Working Directory（工作目錄）

    • 這是你實際進行檔案編輯的地方，也是專案檔案在你的電腦上看得見的實體資料夾。
    • 狀態認定： 所有你新建立或修改過的檔案，在尚未執行 git add 之前，都處於「未追蹤 (Untracked)」或「已修改 (Modified)」狀態。
    • 備註： 專注於你的開發與編輯，你可以隨意修改檔案而不用擔心影響正式的版本紀錄。

  2. 📥 Staging Area（暫存區 / Index）

    • 暫存區是 Git 在正式提交（Commit）之前，用來準備「快照清單」的緩衝區。
    • 進入方式： 使用 git add <檔案名> 指令，將你在工作目錄中的特定變更納入準備清單。
    • 關鍵作用： 它是你決定「哪些變更要包含進下一個版本紀錄 (Commit)」的地方。
    • 備註： Git 允許你只將一個檔案的部分內容（Stage Hunk / Stage Lines） 加入暫存區，實現「Partial Commit」，這對於將大改動拆分成多個邏輯清晰的小提交非常有幫助。（像是一個檔案當中的某一段、某幾行去做Commint）

  3. 🗃️ Git Repository（Git 倉庫 / HEAD）

    • 這是 Git 的核心資料庫，用來儲存專案程式碼、文件、檔案以及所有歷史版本與結構的地方。
    • 進入方式： 當你執行 git commit，暫存區的內容就會被永久寫入這個正式的 Git 版本紀錄中。
    • 狀態認定： 檔案的內容被標記為「已提交 (Committed)」，代表這個版本是穩定且可靠的。
    • 備註：這是版本控制最重要的元素。它儲存了專案所有的歷史快照，並且允許團隊成員安全地在同一個專案上協作，而不會互相覆蓋工作。

這三個區域的循環（修改 → 暫存 → 提交）是所有 Git 操作的基礎！

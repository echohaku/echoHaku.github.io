Git 系列分享 - p.4 你是誰？git config 的配置與層級

October 13, 2025
#Git

在Git 系列分享 - p.3 第一次使用 Git：專案初始化 git init 之後，我們已經成功讓 Git 接管了你的專案。然而，在開始提交 (Commit) 版本之前，Git 需要知道「你是誰」。 而git config 正是用來設定使用者名稱、email、編輯器、顯示風格等參數的工具。這些參數將決定每一次版本紀錄（Commit）時，是誰完成了這項工作。

一、為什麼要設定 git config？（必備設定）

Git 必須將你的身份資訊附加到每一個提交記錄中，以確保版本歷史可被追溯。以下兩個參數是最基本且必須設定的：

參數用途說明範例指令
user.name設定提交記錄上顯示的「使用者名稱」。git config --global user.name "Your Name"
user.email設定提交記錄上顯示的「電子郵件」。git config --global user.email "your.email@example.com"

二、認識 Git 配置的三個層級

git config 的設定可以劃分為三個層級（作用域），它們決定了配置的適用範圍，並且有層級優先順序：

  • System（系統層級）
    • 適用於整台電腦的所有使用者。
    • 設定檔位置：git 安裝的資料夾裡，Linux: /etc/gitconfig，Windows: C:\ProgramData\Git\config
    • 指令加上：--system
    • 優先級最低
  • Global（使用者層級） ✅ 最常用
    • 適用於當前作業系統帳號的所有 Git 倉庫。
    • 設定檔位置：電腦登入帳號的目錄底下，Linux: ~/.gitconfig，Windows:C:\User\<帳號>\.gitconfig
    • 指令加上：--global
    • 優先級中
  • Local（倉庫層級）
    • 僅適用於特定的 Git 倉庫。通常是因為有多個帳號（公司用、個人用），所以在個別倉庫去指定用哪個帳號。
    • 設定檔位置：每個.git資料夾內.git/config
    • 預設為 local scope，如果不指定 --global 或 --system，就會套用在這裡，或是用--local。
    • 優先級最高

優先順序與應用情境
Git 會依據 Local → Global → System 的順序來讀取設定。
例如，如果你在 Global 層級設定了個人 Email，但在某個專案（Local 層級）設定了公司 Email，則該專案會優先使用 Local 層級設定的公司 Email。
這樣可以讓你在一台電腦同時去使用兩組或以上的git設定！

如何執行 git config？

  • 用命令列(CLI)執行。
git config --global user.name "指定的使用者名稱"
git config --global user.email "指定的email"
git config --list # 列出所有設定

  • 用視覺化工具 (GUI) 執行 在Sourcetree當中也可以看到類似的設定，預設會先使用 global setting的資料，可以將 「Use global user setting」的勾選清掉，去設定 local config setting。

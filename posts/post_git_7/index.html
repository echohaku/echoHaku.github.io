Git 系列分享 - p.5 檔案追蹤並納入紀錄！（git status、git add）

October 16, 2025
#Git

在Git 系列分享 - p.4 你是誰？git config 的配置與層級完成身分設定後，
我們就要進入Git的核心工作流程了！
今天的主題圍繞在「追蹤變動」與「將變動納入下一個版本」的過程。

一、git status：專案的狀態報告

還記得我們在 Git 系列分享 - p.2 Git 對於修改的文件是怎麼劃分的呢？ 所提到的三個工作區域嗎？
git status 這個指令就是你的「工作區偵測器」，它會掃描你的工作目錄（Working Directory，也就是實體資料夾裡的檔案們），並與 Git 倉庫中的最新版本進行比對，從而確定檔案的狀態：

檔案狀態所在區域顯示於 git status須進行的操作視覺化工具顯示（Sourcetree）
Untracked工作目錄紅色 (Untracked files)需要 git add 納入追蹤Unstaged Files
Modified工作目錄紅色 (Changes not staged for commit)需要 git add 更新暫存區Unstaged Files
Staged暫存區綠色 (Changes to be committed)可以 git commit 正式提交Staged Files
UnmodifiedGit 倉庫不顯示（乾淨狀態）無需任何操作不顯示

當你打開終端機輸入 git status 時，它會清楚地告訴你：

  • 有哪些新檔案還沒被 Git 追蹤？（通常顯示為紅色）
  • 有哪些已追蹤的檔案被修改了，但還沒加入暫存區？（通常顯示為紅色）
  • 有哪些檔案已經在暫存區，準備好要提交了？（通常顯示為綠色）

這個指令是你每次開始或結束工作時，必做的檢查動作。

二、git add：將變動從工作目錄搬到暫存區

git add 是連接「工作目錄」與「暫存區」的橋樑。它的作用是將你在工作目錄中已完成的變更，打包成一個快照，放到暫存區（Staging Area），準備好進行正式提交。

記住： 執行 git add 並不等於提交了版本，它只是準備提交。

指令作用說明
git add <file name>將特定檔案的變動加入暫存區。
git add .將工作目錄下所有「已修改」與「未追蹤」的檔案都加入暫存區。
（最常用，但需謹慎！）
git add -p 或 --patch進階： 允許你選擇性地將一個檔案中的部分變更（Patch/Hunk）加入暫存區。

💡 為什麼需要暫存區？

暫存區的設計是 Git 最精妙的地方。它給你一個機會去審視和組織你的變更：

  • 優勢： 你可以將大量修改拆分成多個邏輯清晰、目標單一的 Commit。
  • 範例： 我可能同時修改了 A 檔案的 Bug 修正 和 B 檔案的 新功能。我會先 git add A 檔案，提交 Bug 修正的 Commit；然後再 git add B 檔案，提交新功能的 Commit。這使得版本歷史更乾淨、更容易追溯。

實戰演練：從 Untracked 到 Staged

首先，我們在專案資料夾中新增一個 readme.md 檔案。
註： readme.md 通常是專案的說明書，會記載專案是什麼、如何使用、注意事項及作者等資訊。看到這個檔名不用緊張，就把它視為商品的說明書即可！

  • 用命令列(CLI)執行。
    • 在sampleFolder增加了readme檔案。
    • 下git status指令後，他偵測到了有一個新的檔案並列出來。
      可以看到檔案會被列在Untracked files區域(紅色)，表示 Git 知道它存在，但尚未開始追蹤。
    • 之後下 git add 指令，告訴 git 我確定這個檔案是已經修改好的，我確定要將這個變更納入下一次的版本快照。之後再下一次git status 就可以看到，檔案狀態變成Changes to be committed區域(綠色)了，這就表示目前的內容已經成功被「暫存」(Staged)，準備好進行提交了！
  • 用視覺化工具 (GUI) 執行
    • 我在sampleFolderForSourcetree增加了readme檔案。
    • 在 Sourcetree 中點選 File Status 頁籤(概念如同git status)，可以看到新增的 readme.md 出現在 Unstaged files 區域（概念如同 Untracked）。點選該檔案，可以在右側看到檔案修改的內容。
    • 確定內容正確後，只要點選檔案旁邊的 + 按鈕（概念如同git add readme.md）（或 Stage all 按鈕，概念如同git add .）。 此時，readme.md 會移動到上方的 Staged files 區域。點選檔案可以看到有顏色的部分表示是此次你確定要提交的內容。（綠色表示新增的部分，紅色表示刪除。）
      視覺化工具可以讓你更直觀地看到，哪些檔案已經被打包，等待提交。

恭喜你！檔案已經順利進入暫存區。接下來，我們將進入最後一步：使用 git commit 為這個變更留下一個永久、有意義的紀錄，並了解如何使用 .gitignore 忽略不想追蹤的檔案。

Git 系列分享 - p.1 Git是什麼？

October 1, 2025
#Git

如同Git 系列分享 - p.0 版本控制是什麼？有提及的，Git 是一種開源的「分散式版本控制系統（DVCS）」，它專門設計來高效且可靠地處理專案中檔案的每一次變動。

Git 的核心優勢

Git 之所以能在眾多版本控制工具中脫穎而出，成為現代數位工作者的標準配備，主要歸功於其以下特性：

  • 分散式架構 (High Redundancy)：
    • 每個使用者都有專案完整的歷史紀錄。即使中央伺服器故障，你仍可以從任何一位使用者端恢復資料。這讓系統的容錯率極高，能有效避免因單點故障而導致版本控制功能停擺。
  • 快照機制 (Snapshots)：
    • 機制： Git 紀錄的不是「檔案的差異」，而是將整個專案的狀態拍攝「快照」並儲存下來。(註：這個機制非常精妙，未來有機會的話我們會用一整篇文章來深入描述！）
    • 優勢： 這使得版本切換和歷史紀錄查詢極為迅速，效率遠高於傳統的集中式系統。
  • 開源與免費 (Free & Open Source)：
    • 任何人都可以免費使用 Git，這也是它能迅速普及的重要原因之一。
  • 強大的分支管理 (Branching)：
    • Git 的分支（Branch）操作非常輕便、快速且不佔資源，讓開發者可以在不影響主線的情況下，安全地進行功能開發或實驗。

Git 的挑戰與考量（劣勢）

雖然 Git 功能強大，但在某些情況下，它也會帶來一些挑戰：

  • 學習曲線陡峭：

    • Git 的底層概念（例如：Stage, Commit, Branch, Rebase 等）相對抽象，對於初學者來說，學習門檻確實較高。

  • 不擅長處理大型二進位檔案 (Large Binary Files)：

    • Git 的快照機制是針對文字檔案進行高效壓縮。對於大型的二進位檔案（例如：高畫質圖檔、影音檔、大型資料庫備份），Git 會完整儲存每一個版本的快照，導致版本庫（Repository）會快速膨脹，佔用大量空間，且同步速度變慢。（解決方案：這個問題通常會透過搭配 Git LFS (Large File Storage) 等擴展工具來解決。）

  • 部分操作的破壞性：

    • 某些進階操作（如 rebase 或 reset）雖然功能強大，但如果操作不當，可能會修改歷史紀錄，導致與團隊成員的版本衝突，甚至遺失資料，需要謹慎使用。

Git 的使用途徑

對於新手而言，Git 的學習門檻確實相對較高，這主要體現在它複雜的指令和概念上。不過，你可以選擇以下幾種方式來使用 Git：

  • 終端機／命令列 (Terminal / CLI)：

    • 這是使用 Git 最標準且最有效的方式，可以執行所有功能。
    • 在 Linux 和 macOS 系統上使用 Terminal；在 Windows 系統上則常使用 Git Bash 或 命令提示字元 (CMD)。
    • 註：Vim 是 Git 預設的編輯器，當你需要輸入提交訊息（Commit Message）時會跳出。熟悉 Vim 的簡單操作（如 i 進入編輯、Esc 退出、:wq 存檔並離開）對初學者會有幫助。

  • 視覺化圖形介面工具 (GUI)：

    • 對於不習慣指令操作的人，可以使用 Sourcetree、GitKraken 或 VS Code 內建的 Git 介面等工具。
    • 優點： 這些工具讓你透過點擊畫面就能完成大部分常用的 Git 操作（如提交、切換分支、合併等），能大幅降低初學者的學習壓力。

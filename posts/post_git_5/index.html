Git 系列分享 - p.3 第一次使用 Git：專案初始化 git init

October 8, 2025
#Git

在Git 系列分享Day 2 - Git 對於修改的文件是怎麼劃分的呢？ 我們了解了 Git 如何將專案劃分為工作目錄、暫存區和 Git 倉庫這三個區域。現在，讓我們開始實戰，學習如何讓 Git 開始管理你的專案！

啟動版本控制：git init 的奧秘

要讓 Git 接管一個資料夾，使其具備版本控制能力，我們需要做的第一步就是初始化 (Initialize) 專案。這個動作是透過 git init 指令來完成的。

當你執行 git init 之後，Git 就會在這個資料夾內建立它的「資料庫」，專案才正式進入 Git 的管理流程。

git init 實際做了什麼？

  • 建立核心資料庫：.git 資料夾

    • Git 會在你的專案資料夾內建立一個隱藏的 .git 資料夾。
    • 這個資料夾就是你的 Local Repository (本地倉庫)，它包含了所有 Git 運作所需的檔案和資料結構，例如：所有版本紀錄 (Commit History)、分支資訊 (Branches)、配置設定、以及你之前提過的 暫存區 (Index) 資訊等。
    • 重要性： 這是 Git 儲存所有歷史快照的核心資料庫。請勿手動修改或刪除此資料夾內的任何檔案！

  • 初始化主分支：

    • Git 會自動建立一個預設的主線分支，傳統上稱為 master，但現在許多新的 Git 版本和平台會將其預設為 main。這就是你專案的起點。

如何執行 git init？

  • 用命令列(CLI)執行。

    1. 開啟終端機（Windows的cmd或git bash、macOS/Linux的 Terminal）
    2. cd 到要管理的資料夾位置
    3. 輸入git init
    4. 點下 enter，你會看到類似Initialized empty Git repository in /path/to/your/project/.git/的回覆，這表示 Git 已經成功初始化。

  • 視覺化工具 (GUI) 執行 若你不習慣使用指令，可以透過視覺化工具來操作。這對於害怕代碼的初學者來說是非常友好的替代方案。下方以Sourcetree為例

    1. 在 Sourcetree 介面中，點擊 + New Tab（或直接點擊新增按鈕）。
    2. 選擇 Create 選項，表示要建立一個新的 Repository。
    3. 選擇指定的資料夾路徑作為專案目的地。
    4. 點擊 Create，工具就會在背景執行 git init 相同的功能，完成資料夾的初始化。

git init 就像是為你的專案安裝了「版本時光機」的核心引擎。完成初始化後，你的專案才算是正式成為一個 Git 專案！下一頁我們要來說設定你的Git了呦！請參考 Git 系列分享 - p.4 你是誰？git config 的配置與層級

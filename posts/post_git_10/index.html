Git 系列分享 - p.8 Git 的核心力量 - 分支 (Branch)

October 23, 2025

在了解Git 系列分享 - p.7 版本控制的篩選器 - .gitignore 的使用與規則之後，其實你已經可以掌握了Git的基礎單人工作流程了！然而，Git 之所以能成為團隊協作的標準工具，全賴於它最核心且強大的功能：分支（Branch）。

一、git branch 的概念：什麼是分支？

分支的目的是什麼呢？分支的核心目的是為了解決同時進行多項修改時互相干擾的問題，確保主線版本是穩定的。

在傳統世界裡，要想實現分支的概念，可能意味著要複製整個專案資料到另一資料夾上，大家都有這個經歷吧，複製整個專案會花大量的時間，又佔用大量的空間。相較之下，在 Git 的世界裡，分支並不像傳統版本控制系統那樣複雜或佔用大量資源，為什麼呢？

1. 分支是一個輕量級的「指標」

  • 本質： 分支不是一個資料夾的完整複製，它只是一個可移動的指標（Pointer），指向某一個特定的 Commit 紀錄。
  • 預設分支： 當你執行 git init 時，Git 會自動建立第一個分支，通常是 main 或 master，它指向你的第一次 Commit。
  • 運作原理： 由於分支只是一個指標，所以建立、切換、刪除分支的操作都極為快速，幾乎是瞬間完成。

2. HEAD 是你的「當前位置」

  • HEAD： HEAD 是一個特殊的指標，它永遠指向你目前正在工作的分支。
  • 用途： HEAD 指示了下一個 Commit 應該被建立在何處。當新的 Commit 產生時，HEAD 和當前分支指標都會跟著前進。
  • 檢視： 當你輸入 git log 時，你會看到 HEAD -> main，表示你目前位於 main 分支上。

二、分支的使用情境：為何需要分支？

分支最大的價值在於「隔離工作」，讓我們在不影響主線程式碼的條件下，安全地進行開發、測試或實驗。

情境分支名稱建議目的
開發新功能feature/<功能名稱>在獨立的環境下開發完整功能，避免半成品影響到主線（main）穩定版本。
緊急修正 Bughotfix/<問題描述>快速從主線拉出一個分支來修復 Bug，修復完成後馬上合併回主線並發布。
試驗性代碼experiment/<主題>測試一些可能失敗的程式碼或新技術，失敗後可以直接丟棄整個分支，不留痕跡。
多人協作user/<使用者名稱>團隊成員各自擁有自己的分支來進行開發，減少互相干擾和不必要的衝突。

三、Git 核心概念的「交換日記」比喻

我想了很久要用什麼比喻讓大家可以快速理解Git的這些核心元素，最後我認為比較淺顯的方式可能是用「交換日記」來模擬！我們可以將整個版本控制過程想像成一份持續更新的日記本：

Git 概念筆記本比喻說明
工作目錄 (Working Directory)日記本紙張你正在實際寫作和修改內容的地方。
暫存區 (Staging Area)用鉛筆打稿你用鉛筆寫下，隨時可以擦掉重寫，代表這些變動已準備好，但還沒定稿。
Commit定稿並蓋上日期章用原子筆將鉛筆稿定稿，寫上你的名字和日期，成為一頁不可修改的「歷史頁面」。
Branch (分支)不同顏色的便利貼一張輕薄的便利貼，貼在某個歷史頁面（Commit）上，標示一條獨立的工作線（如：聊Ａ時事用一個顏色、Ｂ時事用另一個顏色）。
HEAD書籤這個書籤永遠插在你當前正在看/使用的那張「便利貼」（分支）上，指示你下一個 Commit 要釘在哪裡(下一個要寫的內容要寫在哪一張便利貼上)。

四、核心分支指令 (git branch & git switch)

以下是你在日常開發中最常使用的分支相關指令：

指令作用說明備註
git branch列出所有本地分支。當前所在的分支會以 * 號標記。
git branch <name>建立一個新的分支，但不會切換過去。該分支會指向當前 Commit。
git switch <name>切換到指定的分支。
git switch -c <name>建立並切換到新的分支。等同於 git branch <name> 加上 git switch <name>。
git checkout -b <name>建立並切換到指定分支
git branch -d <name>刪除指定分支（安全模式）。只能刪除已經合併到其他分支（例如主線）的分支，避免丟失未整合的變更。
git branch -D <name>強制刪除指定分支。即使分支內容未合併，也會被刪除（請謹慎使用）。

實戰範例

  • 用命令列(CLI)執行。
    1. 先透過git branch看到目前有哪些 branch的存在，其中 帶有*號且綠色 字樣的就是你目前在的分支(HEAD所在之處)
    2. 執行 git checkout -b feat/featA 去建立並將HEAD移到featA上，在執行一次git branch可以看到改變！
  • 用視覺化工具 (GUI) 執行 (Sourcetree)：
    1. 在指定的commit點上按下右鍵，選擇Branch
    2. 輸入 Branch名稱，若有勾選Checkout New Branch則會自動 checkout 那新Branch上
    3. 可以看到左邊的Branches 就如同執行 git branch一樣，列出所有分支。右側可以看到目前feat/featA和master都是在同一個commit上的

五、值得分享的知識點

1. git switch 與 git checkout 的區別

在舊版本的 Git 中，切換分支的指令是 git checkout。這個指令功能太過強大，同時負責「切換分支」和「還原檔案」，容易造成誤解。

  • git switch： 僅用於切換分支。這是 Git 官方在 2.23 版本後推薦的新指令，功能單一且安全。
  • git restore： 用於還原檔案（例如從暫存區撤銷變更）。
  • 建議： 為了清晰和安全，建議新學習 Git 的使用者都改用 git switch 和 git restore。

2. 分支操作為什麼如此快速？

Git 之所以能如此高效，是因為當你建立一個分支時，它並沒有複製專案中的所有檔案，而是：

  • 只建立一個 40 位元的 SHA-1 雜湊值指標。(很小很小很小就對了！)
  • 該指標指向你當前的 Commit 物件。 這使得分支的儲存空間佔用極小，你可以在幾秒鐘內建立數百個分支，這對於大型專案的測試和開發來說至關重要。

小結

分支是 Git 協作與隔離的基石。學會了分支，你就能安全地進行任何改動。

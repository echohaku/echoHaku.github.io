使用postgreSQL時數字類型的 ID 該如何使用 sequence 功能

January 20, 2026
#Database
#Java

Sequence

  • PostgreSQL 是「原生支援 Sequence」的資料庫，JPA / Hibernate 與它的整合是非常完善
  • 在 persist/save 當下，你就一定能拿到 ID
  • create table時，若ID是數字自動增加的話，
    直接CREATE SEQUENCE xxx_id_seq START WITH 1 INCREMENT BY 1;
  • TABLE 當中 id 設定 id INTEGER NOT NULL DEFAULT nextval('order_id_seq'),
  • JAVA Entity 設定 @GeneratedValue 時搭配使用
    @GeneratedValue(strategy = GenerationType.SEQUENCE, generator = "order_seq")
    @SequenceGenerator(name = "order_seq",sequenceName = "order_id_seq",allocationSize = 1)
項目說明
ID 產生時機INSERT 前就能取得 ID
Hibernate 行為先 nextval(sequence)，再 INSERT
是否支援 batch insert支援
交易回滾Sequence 不會回滾（正常行為）
效能⭐⭐⭐⭐⭐（最佳）

搭配 @GeneratedValue 該如何使用

StrategyPostgreSQL是否推薦原因
SEQUENCE✅ 原生⭐⭐⭐⭐⭐效能最佳、可預測
IDENTITY✅⭐⭐⭐易用但效能較差
AUTO⚠️⭐⭐行為不明確
TABLE❌❌效能差
UUID❌❌不符合 Integer

總結

PostgreSQL 中 Integer 型別的 Primary Key，
正確做法是：SEQUENCE + DEFAULT nextval() + GenerationType.SEQUENCE。

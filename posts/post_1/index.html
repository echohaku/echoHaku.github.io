使用 Hugo GitHub 建立個人部落格

May 16, 2025

起源與目的

在 2025 年 5 月的這段時間，我正經歷一段迷茫期。思考了許久，試著找出一種方式，能讓自己對所理解與學習的知識更加有自信，也希望能夠逐步建立起屬於自己的知識體系。最終，我決定開始寫部落格，分享自己學習到的內容、遇到的問題、解決的過程，以及個人的思考與心得。

雖然不知道這樣的分享能夠持續多久，但我希望能盡可能長久地維持這個習慣，把自己腦中吸收的知識透過文字整理出來。經過一番資料查找後，我決定透過 Hugo 搭配 GitHub 來實踐這個想法。

選擇這個方法，是因為它不需要太多繁瑣的 coding，也不必花太多時間糾結於主題與配色的選擇，對我來說是個簡單又實用的開始。

以下是我所使用的建置方式與流程。

實作

Stage 1. 工具準備工具準備

  1. 安裝 Git (可選擇搭配 視覺化GUI工具)
  2. 安裝 Hugo
  3. 註冊 GitHub 帳號
  4. 選定一個 Hugo theme

Stage 2. 建立並設定Hugo專案

  1. 先在GitHub上創建兩個repository，並將其clone到地端
    一個取名為blogger，用來儲存網站原始碼與內容（文章、圖片等）
    一個則是取名為 username.github.io，用以儲存hugo為我們產生的靜態內容

  2. 初始化Hugo專案

     cd blogger
 hugo new site sitename

    此命令會在 blogger/sitename 下建立 Hugo 專案。會看到 /blogger/sitename/ 底下許多的子folder

  3. 安裝主題 (theme) 在 hugo themes 上選定好一個主題，可以看到 “download” 主題的按鈕，通常可以連結到該theme的github page

     cd sitename
 git submodule add 該theme的git themes/rusty-typewriter

    這個指令的目的是把該主題鑲嵌的概念到你的repository上，當這個主題的作者有所變動，我們這邊也可以有最新的commit結果，可以確認 .gitmodules 的檔案是否有正確的新增。 修改 sitename/hugo.toml

     baseURL = https://username.github.io/
 languageCode = "en-us"
 title = 'blog的名稱'
 theme = 'theme name'

  4. 預覽部落格

    cd sitename
hugo server

    會看到 blogger/sitename/public的資料夾產生，這裡就是所有靜態檔案存在的地方。 瀏覽 http://localhost:1313 確認網站預覽狀況。

  5. 撰寫內容

    cd sitename
hugo new posts/postname.md

    這會根據 /sitename/archetypes/default.md 建立文章，實際產生的內容檔案位於 sitename/content/posts/postname.md，可修改其中的 title、date、draft 與文章內容。

Stage 3. GitHub Actions 自動部屬

  1. 建立GitHub Token
    透過 GitHub Token ，按下「generate new token(classic)」去產生token，權限設定repo、workflow。建立後務必複製紀錄，因為此token在離開畫面後將不可再次查看。
  2. 設定 Secrets
    前往 blogger repository -> Settings -> Secrets and Varaibale -> Actions -> New repository secret -> name: DEPLOY_TOKEN，將先前複製的Token放在Secrets當中。
  3. 設定 GitHub Actions Workflow
    建立 blogger/.github/workflows/deploy.yml
     name: Deploy Hugo to GitHub Pages Repo

 on:
 push:
     branches:
     - main

 jobs:
 deploy:
     runs-on: ubuntu-latest

     steps:
     - name: Checkout source repo
     uses: actions/checkout@v4
     with:
         submodules: recursive

     - name: Setup Hugo
     uses: peaceiris/actions-hugo@v2
     with:
         hugo-version: 'latest'
         extended: true

     - name: Build the site
     run: cd $sitename && hugo --minify

     - name: Deploy to $username.github.io repo
     uses: peaceiris/actions-gh-pages@v3
     with:
         personal_token: ${{ secrets.DEPLOY_TOKEN }}
         external_repository: $username/$username.github.io
         publish_dir: ./$sitename/public
         publish_branch: main
         user_name: $username
         user_email: $usermail
         commit_message: ${{ github.commit_message || 'Deploy from blogger repo' }}
    記得將所有$開頭的變數請自動換成自己的直接取代（包括$username, $sitename, $usermail）
    另外with: submodules: recursive => 去拉.gitmodules裡面所設定的其他git repository資訊
  4. 忽略public 目錄版本控制
    在blogger/.gitignore加入/sitename/public/
    避免將 /public 目錄內容加入版本控制，讓它完全由 GitHub Actions 負責處理
  5. Commit & Push 原始碼
    將原始碼（含設定、文章）推送至 GitHub：
    git add .
git commit -m "init hugo blog"
git push origin main

Stage 4. 驗證部屬

  1. 前往 GitHub → blogger → Actions 檢查工作流程是否成功執行
  2. 開啟 https://username.github.io/，應可看到部落格成功部署

Recap sitename架構

blogger/
├── sitename/       ← Hugo 專案實際位置
│   ├── hugo.toml   ← (config.toml)
│   ├── content/
│   └── ...
├── .github/
│   └── workflows/
│       └── deploy.yml
├── .git
├── .gitmodules
└── .gitignore

心得

在實際動手操作的過程中，我遇到的錯誤比預期中多出許多，花費的時間也自然超出了原本的估計。雖然整體過程中不免感到挫折，但我很慶幸自己沒有因此放棄，而是一步一步地解決問題，最終順利完成了部落格的建立。🍻

萬事起頭難，尤其是在從零開始學習與實作的情況下，遇到預料之外的困難與錯誤幾乎是必然的。不過，這次的經驗讓我深刻體會到：犯錯其實沒有想像中那麼可怕。只要願意冷靜面對、查找資料並持續嘗試，問題終究是能被一一克服的。

過程中，大多數的錯誤其實都來自一些細節——像是在錯誤的位置執行了正確的指令，或是整體架構理解不夠清晰。這些錯誤往往令人沮喪，但只要靜下心來、仔細觀察，就能找出問題所在並修正它。

此外，在實作過程中我也發現，不同的 Hugo 主題都有其獨特的功能設計，各有巧思，其實非常值得深入研究。不過，現階段我已經達成了對這個部落格初始化階段的期望，因此先在這裡畫下一個小小的句點。未來若有更多時間與精力，我相信自己會更深入探索並拓展這個部落格的可能性！

我相信，最困難的一關已經跨越。也期待在下一篇文章中，能繼續與大家分享更多學習過程與心得！😄

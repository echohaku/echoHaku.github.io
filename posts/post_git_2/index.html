Git 系列分享 - p.0 版本控制是什麼？

September 25, 2025
#Git

版本控制到底是什麼？到底可以幫助我們解決什麼問題？

「版本控制」

在正式認識進入Git 系列文章 之前，我們必須先理解「版本控制」這個核心概念。

過去，當我們想備份文件或區分不同版本時，通常會建立一個新資料夾並加上日期，然後將所有檔案複製過去。但這種做法不僅會占用大量空間，也無法快速看出兩個版本之間有哪些差異，更難以追蹤每次修改的目的。這也就是「版本控制」誕生的原因。

版本控制不只是為了工程師而生的！

很多人以為版本控制只對工程師重要，那你就錯了。

想像一下，身為一個文案企劃，你正在修改一份重要文件，結果發現最新版本改得一團糟，想要回到兩天前的雛形文件，卻不知道該怎麼辦。或者你和同事共同編輯一份文件，最終發現兩人的版本內容差異太大，無法順利合併。當你與客戶討論合約內容時，雙方針對合約中的某一行內容各執己見，無法追溯歷史紀錄…這些情境是不是聽起來很熟悉？

不論你是工程師、設計師、小編、業務，這些問題都可能發生在你的日常工作中。Git 其實不只是一個程式碼的版本控制工具，它更像是一台「數位時光機」，能為你處理過的任何檔案（如 Word 文件、設計稿、企劃書等）留下修改紀錄。它讓你不僅能隨時回到過去的任何一個版本，還能有效地與他人協同工作，確保檔案不會混亂。

版本控制是一種強大的專案管理工具，它能比較每次版本差異、追蹤修改依據、並提升協作效率，對於現代數位工作者而言至關重要。

版本控制工具大致核心方式分成兩類

版本控制工具大致核心方式分成兩類：集中式的版本控制系統（Centralized Version Control Sytem）和分散式的版本控制系統（Distributed Version Control System）。 而Git 是其中一種實現「分散式版本控制」的工具！

項目分散式版本控制（DVCS）集中式版本控制（CVCS）
定義每個相關使用者的電腦當中都擁有完整的檔案 Repository 副本，包括所有的歷史紀錄、分支資訊、提交紀錄等等。所有的檔案及版本紀錄都集中於伺服器端，開發者只會擁有目前版本的「副本」，需要連線到伺服器去做大部分的操作。
特點架構和權限控制相對複雜一些，可以離線操作，學習門檻較高。架構簡單，權限控制明確，資料集中，需連線伺服器才可以執行。
版本庫位置每個人都有完整版本庫僅伺服器保有完整版本庫

可離線撰寫程式、commit、切換branch		無法離線 commit / 查看歷史

效能本地操作快須與伺服器通訊，較慢
分支管理快速輕便，支援多分支協作分支較重，複雜

故障容忍性高，資料可從他人 repo 還原低，伺服器若掛掉風險大

適合對象大型團隊、開源專案、需彈性開發小型團隊、流程單純、控管明確
